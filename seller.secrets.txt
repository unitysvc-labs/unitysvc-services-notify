# seller.secrets.txt — the labs seller's secrets for this repo's notification
#                       services (labs/discord-relay, labs/msg-to-discord, …)
#
# Not an example — this is the actual file the labs seller uploads. The values
# are the seller-TEST values (the mock host), which are not real credentials, so
# the file is committed. Each service's offering references these as
# customer_secrets; they must exist in the seller secret store so gateway /
# integration tests resolve them via the ops_customer → seller-secret fallback.
#
# Upload (staging — uses UNITYSVC_SELLER_API_* from ~/.zshrc):
#     usvc_seller secrets upload seller.secrets.txt
#
# Production: set each NAME below as a GitHub repository variable; the
# upload-to-production workflow seeds them into the production secret store
# automatically. Setting the variables is a one-time repo setup, not per-deploy.
#
# Sourceable shell format — `export NAME="value"`; the uploader reads every export.

# ── Apprise — every msg-to-<channel> service (cross-channel) ──

# APPRISE_BASE — OPTIONAL (default: https://apprise.unitysvc.dev)
#   the apprise-api host the gateway POSTs the composed apprise body to, as
#   <base>/notify. For testing, point it at the mock apprise-api server so the
#   discord:// (etc.) transform is validated without delivering to real providers.
export APPRISE_BASE="https://mock.unitysvc.dev/apprise"
